# WordPress Playground

This project uses WordPress Playground in three different contexts.

---

## 1. Playground.net (browser)

The "Try in Playground" badge in the README links to:

```
https://playground.wordpress.net/?blueprint-url=<raw_github_url>/blueprints/blueprint.json
```

**How it works:**

`blueprint.json` contains `writeFile` steps that embed every source file from
`blank-theme` and `wordcamp-org` directly in the JSON. Playground reads the
JSON, writes the files into the virtual WordPress filesystem, then activates the
theme and plugin — no zip download, no external proxy.

**Keeping the badge up to date:**

`blueprint.json` is auto-generated by `scripts/sync-blueprint.js`.
After changing any source file:

```bash
npm run sync-blueprint
git add blueprints/blueprint.json
git commit
```

CI enforces this with `npm run check-blueprint`.

---

## 2. Playground CLI (local, live-reload)

`@wp-playground/cli` runs WordPress Playground as a local Node.js server.
The `blueprint-local.json` uses `mount` steps so the theme and plugin
directories are read directly from the filesystem — no copy, no rebuild.

```bash
npm install          # installs @wp-playground/cli as a dev dependency
npm run playground   # starts local Playground at http://localhost:9400
```

To use `blueprint.json` (writeFile, closer to what playground.net sees):

```bash
npm run playground:web
```

### Mount vs writeFile

| | `blueprint-local.json` (mount) | `blueprint.json` (writeFile) |
|---|---|---|
| File changes | Reflected immediately | Require `npm run sync-blueprint` + restart |
| Suitable for | Active development | Sharing, CI, badge |
| Persistent | No (in-memory) | No (in-memory) |

## 3. GitHub Actions (PR preview)

Every pull request receives a bot comment with a Playground link.

The link uses `?blueprint-url=` pointing to the `blueprint.json` at the PR's
HEAD commit on `raw.githubusercontent.com`. Because the blueprint embeds all
source files directly, the Playground loads exactly what is in the PR — no
artifact proxy, no dist repo.

```
https://playground.wordpress.net/?blueprint-url=https://raw.githubusercontent.com/wordcampbr/wordcampbr/<sha>/blueprints/blueprint.json
```

Installable `.zip` files (for testing in a real WordPress install) are uploaded
as GitHub Actions artifacts with 14-day retention.

## Blueprint schema

Both blueprints follow the official schema:

```json
{
  "$schema": "https://playground.wordpress.net/blueprint-schema.json"
}
```

Validate at: https://playground.wordpress.net/builder/builder.html
